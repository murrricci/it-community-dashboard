# IT Community Dashboard

Дашборд для участников IT-сообществ — отслеживание инженерных практик, RFC-предложений, событий, изменений в стандартах и состава участников.

**Демо:** [murrricci.github.io/it-community-dashboard](https://murrricci.github.io/it-community-dashboard/)

## О проекте

В организации внедряются два стандарта:

- **Стандарт профессий** — профили 6 инженерных специальностей (Backend, Frontend, QA, DevOps, Mobile, Data)
- **Стандарт разработки** — каталог инженерных практик с жизненным циклом внедрения

Дашборд объединяет информацию по обоим стандартам в едином интерфейсе: метрики, RFC, события, участники, история изменений.

## Страницы

| Страница | Описание |
|----------|----------|
| **Дашборд** (`index.html`) | Метрики, последние RFC, ближайшие события, объявления, активные контрибьюторы |
| **Стандарт профессий** (`professions.html`) | 6 профессий с компетенциями, практиками, составом участников и базой знаний |
| **Стандарт разработки** (`practices.html`) | Каталог практик с фильтрами, матрица внедрения, зависимости между практиками |
| **RFC-предложения** (`rfc.html`) | Трекер RFC: черновик → ревью → принято/отклонено, таймлайн статусов, голосование |
| **Календарь событий** (`events.html`) | Список и календарная сетка, фильтры по типу (митап, воркшоп, ревью, демо, ретро) |
| **Участники** (`members.html`) | Директория участников с фильтрами по профессии и роли, лидерборд активности |
| **История изменений** (`changelog.html`) | Таймлайн изменений в стандартах, экспорт в Markdown |
| **Глоссарий** (`glossary.html`) | Термины с алфавитной навигацией и поиском |
| **Онбординг** (`onboarding.html`) | Пошаговый гайд для новых участников, FAQ, шаблон RFC |

## Стек

- **HTML/CSS/JS** — без фреймворков и сборщиков
- **ES-модули** (`type="module"`) — нативная поддержка браузера
- **CSS Custom Properties** — дизайн-токены для цветов, шрифтов, отступов
- **JSON** — данные загружаются из файлов в `data/` через `fetch`
- **GitHub Pages** — автоматический деплой через GitHub Actions

## Возможности

- Тёмная тема в стиле GitHub
- Глобальный поиск по всем сущностям (`⌘K` / `Ctrl+K`)
- Фильтрация по статусу, категории, профессии, роли
- Матрица внедрения практик (профессии × практики)
- Граф зависимостей между практиками
- Календарная сетка и список событий
- Лидерборд активности участников
- Таймлайн статусов RFC
- Экспорт истории изменений в Markdown
- Адаптивная вёрстка (320px — 1280px+)
- Стили для печати
- Deep linking через URL hash
- Персонализация по профессии (localStorage)

## Локальный запуск

```bash
# Клонировать репозиторий
git clone git@github.com:murrricci/it-community-dashboard.git
cd it-community-dashboard

# Запустить локальный сервер (нужен для ES-модулей и fetch)
python3 -m http.server 8000

# Открыть в браузере
open http://localhost:8000
```

Альтернативно — любой статический сервер: `npx serve`, VS Code Live Server и т.д.

## Деплой

Сайт автоматически деплоится на GitHub Pages при пуше в ветку `main`.

Workflow (`.github/workflows/deploy.yml`) использует [peaceiris/actions-gh-pages](https://github.com/peaceiris/actions-gh-pages) для публикации в ветку `gh-pages`.

**Настройка Pages:** Settings → Pages → Source: `Deploy from a branch` → Branch: `gh-pages` / `/ (root)`.

## Данные

Все данные хранятся в JSON-файлах в папке `data/`:

| Файл | Содержимое |
|------|-----------|
| `professions.json` | 6 профессий с компетенциями и составом |
| `practices.json` | Инженерные практики с категориями и статусами |
| `rfcs.json` | RFC-предложения с историей и голосованием |
| `events.json` | События сообществ |
| `members.json` | Участники с ролями и контрибуциями |
| `changelog.json` | История изменений в стандартах |
| `glossary.json` | Термины и определения |
| `announcements.json` | Объявления и новости |
| `resources.json` | База знаний (ссылки) |
| `adoption-matrix.json` | Матрица внедрения практик |
| `practice-dependencies.json` | Зависимости между практиками |

Для обновления контента достаточно отредактировать JSON-файлы и запушить изменения.
